# Task Completion Checklist

## After Making Changes
1. **Preview locally**: `mkdocs serve` to verify changes render correctly
2. **Check math**: Ensure MathJax formulas display properly
3. **Verify links**: Check internal documentation links work
4. **Build test**: `mkdocs build` to ensure no errors

## Before Committing
- Use branches for changes (user preference)
- Create pull request at end (user preference)
- No unauthorized changes without asking

## Deployment
- `mkdocs gh-deploy` pushes to GitHub Pages
